Based on current application structure, we can identify the following microservices:

- Person Services: Handles operations related to persons.
- Location Services: Manages location data and related functionalities.
- Connection Services: Handles connections and queries based on location proximity.

Frontend communicate to Backend services (Connection services and Person services), using a RESTFul API

Connection services communicate to Person services use a RESTFul API to retrieve all persons information

Create new Location and Person are refactored to use Kafka and GRPC.
